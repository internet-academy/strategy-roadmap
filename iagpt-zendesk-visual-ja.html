4つの主要な課題

1. ベンダー依存

IAはナレッジプラットフォームを管理していない。Zendeskが値上げや機能変更を行ってもIAには交渉力がない。

2. 継続コスト（~¥400〜480万/年）

Suite Team、4エージェント、$55/エージェント/月。実際の使用量に関わらず毎年発生するコスト。

3. ナレッジの分散

サポートナレッジが3システムに分散。スタッフは週2時間を検索に無駄に — これは~¥830万/年のスタッフ時間損失（4名×2h×52週×¥2万/h）。

4. AI統合パスなし

ZendeskのナレッジはIAGPT AIワークフローに使えない。スタッフはサポートナレッジに基づいたAI支援回答を得られない。

各課題の解決方法

ベンダー依存 → IA管理

ナレッジがIAGPTに移行され、IAが所有・管理。ベンダーロックイン解消。

継続コスト → 排除（フェーズ4後）

Zendesk解約時（目標: 2027年2月）、~¥400〜480万/年の支出が停止。フェーズ2はこの決定の基盤を構築。

分散 → 統一検索

スタッフはIAGPTのみ検索 — Zendesk + Confluenceナレッジが1箇所に。2026年7月から直ちに週2時間/人を回収。

AIパスなし → AI支援回答

IAGPTがサポートナレッジに基づくAI支援回答を提供。スタッフはより良い回答をより速く取得、品質は人間がレビュー。

フェーズ2のアプローチ: 2つのトラック

フェーズ2A — ナレッジ取り込み

  • 58の承認済みZendesk記事をスタッフ専用IAGPTに取り込み
  • サポート/スクールデスクが各記事の正確性をレビュー
  • スタッフがIAGPTでサポート回答を検索・利用可能
  • 推定工数: ~87h（レビュー + 取り込み + テスト）

フェーズ2B — 置き換え準備

  • チャット、チケット、エスカレーション、レポート、運用ワークフローを特定
  • 各Zendesk機能を分類: 「IAGPTに置き換え」または「Zendeskに維持」
  • ギャップ分析 — 解約前に構築すべきものを特定
  • フェーズ2では置き換え作業は行わない — インベントリと計画のみ
